FireSim Basics¶

FireSim is a cycle-accurate, FPGA-based hardware emulation platform. FireSim supports many of the features of commercial hardware emulators, like full-visibility waveform capture, on both readily available FPGA development boards and multi-FPGA prototyping systems, like Siemens Primo.

Background/Terminology¶

FireSim Manager (firesim)¶

This program (firesim) automates the work required to launch FPGA simulations. Most users will only have to interact with the manager most of the time.

Emulation vs Simulation¶

Throughout these docs we use the terms “emulation” and “simulation” interchangeably. A bias towards the latter belies FireSim’s academic roots — not to mention it’s in the name!

Metasimulation¶

Refers to use of a conventional RTL simulator (e.g., VCS) to host a FireSim, in lieu of a physical FPGA.

Computer Disambiguation¶

To differentiate between the computers being simulated and the computers doing the simulating, we also define:

Target¶

The system (a DUT and its harness) under simulation. Generally, a group of one or more RISC-V microprocessors connected to a series of FireSim bridges (i.e., transactors) to model I/O.

Host¶

The computer(s) executing the FireSim.

In these docs, we frequently prefix words with these terms. For example, target-software (RISC-V code executing on the simulated system) vs host-software (software executing on the x86 in support of the simulation); target-clock (a clock in the simulated SoC) vs host-clock (the primary free-running clock used in FireSim); target-cycles (simulated cycles) vs host-cycles (raw, free-running FPGA cycles).

Rhodium¶

The FIRRTL compiler used by FireSim to convert target RTL into a decoupled simulator, based on LLVM’s CIRCT.

Key Files / Data Structures¶

config_runtime¶

The main runtime configuration used to govern a single FireSim run.

config_hwdb¶

A representation of one or more built simulators, composed of URLs or paths components of the model (e.g., FPGA bitstreams).

Workload¶

A representation of a RISC-V software application in addition to supporting firmware and system software (if applicable), be executed over one or more FireSims.
